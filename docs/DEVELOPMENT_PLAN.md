# ARCANA GRID Development Plan

この計画は、今後のCodex指示を短くするための開発順序メモです。実装済み仕様の詳細は `docs/GAME_RULES.md`、現在の状態は `docs/CURRENT_STATUS.md` を正とします。

## 原則

- ルールエンジンをUI・Next.js・Supabaseから分離する。
- UIはモバイルファースト、iPhone縦画面優先で進める。
- オンライン対戦はサーバー権威型にし、相手の伏せカード情報を相手クライアントへ送らない。
- 未確定仕様は実装前にTaskへ分割し、既存仕様を勝手に変更しない。
