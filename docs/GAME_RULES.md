# ARCANA GRID Game Rules

この文書は、現在のコードで実装済みの確定ルールだけをまとめます。未実装または未確定の将来機能は末尾へ分離します。

## 対象ルールセット

- 現在の実装対象は `gameMode: "tactical_duel"`。
- ルールバージョンは `tactical_duel.v1`。
- 盤面は8×8。
- 各プレイヤーは初期配置6体、リザーブ2体。
- 初期配置範囲は自陣側2行。
- 旗の最大ダメージは3。

## 対戦状態

- Matchは `waiting` / `setup` / `active` / `finished` / `aborted` のphaseを持つ。
- 対戦開始前のsetupでは `currentTurnPlayerId` は `null`、`turnNumber` は0。
- 対戦開始時に明示された先手が `currentTurnPlayerId` になり、`turnNumber` は1になる。
- 成功した行動は `stateVersion` を進め、イベントを返す。
- 行動は `expectedStateVersion` で古い状態への適用を拒否する。

## 初期配置とリザーブ

- 各プレイヤーは6体を自陣初期配置範囲へ配置する。
- 各プレイヤーは2体をリザーブとして保持する。
- 初期配置・リザーブ数はルール設定値で検証される。
- 初期配置先は盤内、自陣初期配置範囲内、旗エリア外、他ユニットと重複なしである必要がある。
- リザーブ配備は、自分の盤上ユニット数が初期配置数未満で、リザーブユニットが残っている場合に可能。
- リザーブ配備先は自陣初期配置範囲内、旗エリア外、空きマスである必要がある。
- リザーブ配備されたユニットは `status: "board"` になり、`currentDefense` はカードの `baseDefense` になる。

## 盤面・旗エリア

- 座標は内部的に `row` / `col` で扱う。
- north側の旗エリアは最上段中央2マス、south側の旗エリアは最下段中央2マス。
- 8×8ではnorthが `(0,3)` / `(0,4)`、southが `(7,3)` / `(7,4)`。
- 旗エリアにユニットを置く初期配置・リザーブ配備はできない。

## 移動

- カードは `movementType` と `movementRule` を持つ。
- 実装済みの移動種別は4種類: `orthogonal`、`diagonal`、`adjacent`、`special_offset`。
- ルール計算としては `line` と `offset` のmovementRuleを扱う。
- `line` は指定方向へ最大距離まで進む。最大距離が `null` の場合は盤端まで進む。
- 味方マスは通過可能だが停止不可。
- 敵マスは攻撃対象になり、敵を越えて進めない。
- `offset` は指定された到達先候補を評価し、盤外と味方マスを除外する。敵マスは攻撃対象になる。
- defeatedユニット、盤外位置、盤上にいないユニットの移動は拒否される。

## 表示姿勢とcurrentDefense

- ユニットの姿勢は `attack` または `defense`。
- 移動、旗攻撃、リザーブ配備では次の姿勢を指定する。
- `currentDefense` はユニットの現在防御値。
- 戦闘で減少し、0未満には丸められる。
- defeatedになったユニットは `position: null`、`currentDefense: 0` になる。

## 伏せカードと公開条件

- 所有者は自分のユニットを常に `owner_full` として見る。
- 相手から未公開のユニットは `hidden` として扱い、カード詳細ではなく `cardBackKey` を返す。
- 公開済みユニットは `revealed` としてカード詳細を返す。
- 初回移動が確定したユニットは相手へ公開される。
- 攻撃された防御側ユニットは攻撃者へ公開される。
- 現在の設定では公開状態は対戦終了まで維持される。

## 戦闘

- 戦闘は異なる所有者の、盤上にいる2体のユニット間でのみ解決する。
- 攻撃側・防御側の攻撃力とcurrentDefenseは、0以上の有限数である必要がある。
- 防御側が攻撃表示の場合:
  - 攻撃力が同値なら両方defeated。
  - 攻撃側の攻撃力が高い場合、防御側はdefeated。攻撃側は防御側の攻撃力分だけcurrentDefenseを失い、0以下なら攻撃側もdefeated。生存時は目的地へ移動する。
  - 防御側の攻撃力が高い場合、攻撃側はdefeated。防御側は攻撃側の攻撃力分だけcurrentDefenseを失い、0以下なら防御側もdefeated。生存時は目的地に残る。
- 防御側が防御表示の場合:
  - 防御側のcurrentDefenseから攻撃側の攻撃力を引く。
  - 0以下なら防御側はdefeatedし、攻撃側が目的地へ移動する。
  - 1以上残るなら防御側は目的地に残り、攻撃側は元の位置に戻る。

## 旗攻撃

- `ATTACK_FLAG` はactive中の自分のターンに実行する。
- 攻撃ユニットは合法移動で相手旗エリアの対象座標へ到達できる必要がある。
- 旗エリアにユニットが存在する状態では旗攻撃を拒否する。
- 旗への攻撃は通常1ダメージを与える。
- 旗ダメージは最大値3で止まる。
- 3回目の旗ダメージ到達で `flag_destroyed` による終了条件を満たす。

## 勝敗・終了条件

- 全滅: プレイヤー所有ユニットがすべて `defeated` になると相手が勝利する。
- 旗破壊: プレイヤーの旗ダメージが最大値に到達すると相手が勝利する。
- 投了: `CONCEDE_MATCH` により相手が勝利する。
- 勝敗確定時はphaseが `finished` になり、`winnerPlayerId` と `winReason` が設定される。
- 両者が同時に敗北条件を満たす曖昧な状態はエラーとして扱う。

## 対戦開始から終了まで

1. setup状態のMatchを用意する。
2. 両プレイヤーが初期配置6体とリザーブ2体を確定する。
3. 明示された先手で対戦を開始し、activeへ遷移する。
4. 現在手番のプレイヤーが移動・戦闘・旗攻撃・リザーブ配備・投了のいずれかを行う。
5. 行動成功時にstateVersionとターンが進む。勝敗条件を満たした場合はfinishedへ遷移する。
6. finished後の通常行動は拒否される。

- Supabaseのスキーマ、migration、RLS、サーバー側ルール実行。
- オンライン2人対戦の作成・参加・同期・再接続。
- カードマスタ、カード画像管理、カード追加ワークフロー。
- Vercel公開設定と本番運用確認。
- disconnect勝利ルールの確定。型には暫定理由があるが、詳細ルールは未確定。
